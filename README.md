# streamlit_PHDcharts

位置的頭蓋変形に関する症例データを Streamlit 上で可視化し、治療前後の変化や患者分布を確認するためのアプリです。外部 API から取得したデータをもとに、月齢、頭囲、短頭率、CA、CVAI、CI などの指標を集計・表示します。

## 概要

このリポジトリは、`app.py` をエントリーポイントとする単一構成の Streamlit アプリです。アプリ起動時に `st.secrets["API_URL"]` からデータ取得先を読み込み、患者経過データ、ヘルメット情報、患者数情報を取り込んで可視化します。

確認できたデータ処理の主な流れ:

- API から `経過`、`ヘルメット`、`患者数` の各データを取得
- `ダミーID` が欠損または空文字のレコードを除外
- 各種計測値を数値化し、解析用データフレームを作成
- 治療前後、経過観察、ヘルメット種類、クリニック別にデータを集計
- Plotly グラフと表で分布、推移、比較結果を表示

## 主な機能

- 初診患者数と治療患者数の時系列表示
- 指標ごとのヒストグラム表示
- ヘルメット種類の構成比表示
- 治療前月齢の構成比表示
- 治療前後の各指標の変化の可視化
- ヘルメット別比較
- 経過観察群の推移表示
- クリニック間比較

コード上で確認できる比較対象の主な指標:

- `頭囲`
- `短頭率`
- `前頭部対称率`
- `後頭部対称率`
- `CA`
- `CVAI`
- `CI`

## セットアップ

### 1. 依存関係のインストール

```bash
pip install -r requirements.txt
```

`requirements.txt` に含まれている主な依存関係:

- `streamlit`
- `matplotlib`
- `numpy`
- `pandas`
- `plotly`
- `scipy`

補足:

- `app.py` では `requests` も利用されています。環境によっては追加インストールが必要です。

```bash
pip install requests
```

### 2. Streamlit secrets の設定

このアプリは `st.secrets["API_URL"]` を参照します。`.streamlit/secrets.toml` などに、データ取得先 API を設定してください。

例:

```toml
API_URL = "https://example.com/api"
```

注意:

- 実際の API レスポンスには、少なくとも `経過`、`ヘルメット`、`患者数` の各キーが含まれている前提で処理されています。
- README では秘密情報や実運用 URL は記載していません。

## 実行方法

ローカル実行:

```bash
streamlit run app.py
```

`Procfile` では次のコマンドで起動する設定になっています。

```bash
streamlit run app.py --server.port $PORT
```

## アプリ内で扱っている分類・集計

コード上では、治療前後の重症度分類や比較のために以下のような整理が行われています。

- PSR / ASR レベル分類
- CA / CVAI 重症度分類
- 短頭症分類
- 治療期間の算出
- ダミー ID の接頭辞によるクリニック分類

確認できたクリニック分類:

- `T` で始まる ID: 日本橋
- `K` で始まる ID: 関西
- `H` で始まる ID: 表参道
- `F` で始まる ID: 福岡

また、コード上ではヘルメット種別として少なくとも以下が使われています。

- アイメット
- クルム
- クルムフィット
- 経過観察
